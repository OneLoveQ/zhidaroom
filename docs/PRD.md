# 智答课堂 AI MVP 产品需求文档

## 1. 产品定位

智答课堂 AI 是面向中国中小学课堂的纸卡式即时反馈与 AI 教学诊断系统。

MVP 版本只验证一个核心闭环：

教师创建题目 -> 大屏展示 -> 学生举卡 -> 教师扫码 -> 实时统计 -> AI 讲评建议 -> 课后报告。

首个切入场景为“出口检测”：每节课最后 3 分钟，教师用 1 到 3 道题快速判断全班掌握情况。

## 2. 目标用户

| 角色 | MVP 核心需求 |
| --- | --- |
| 任课教师 | 快速创建检测活动，采集全班答案，查看统计和讲评建议 |
| 学生 | 使用纸质答题卡参与课堂，不需要设备和账号 |
| 学校管理员 | 后续管理教师、班级、题库和权限，MVP 仅预留角色 |

## 3. MVP 范围

### 3.1 必须实现

- 教师登录。
- 班级创建。
- 学生导入。
- 答题卡编号绑定。
- 单选题创建。
- 出口检测活动创建。
- 课堂大屏展示题目和匿名统计。
- 扫码结果批量提交接口。
- 实时答题统计。
- 基础课堂报告。
- AI 生成单选题。
- AI 错因分析和讲评建议。

### 3.2 暂不实现

- 家长端。
- 学生 App。
- 教师排名。
- 区域教研平台。
- 主观题自动批改。
- 长期学生画像。
- 复杂知识图谱。
- 多校数据对比。

## 4. 核心流程

1. 教师登录系统。
2. 教师创建班级。
3. 教师导入学生名单。
4. 教师为学生绑定答题卡编号。
5. 教师创建单选题。
6. 教师创建出口检测活动。
7. 大屏展示当前题目。
8. 学生旋转纸卡选择 A/B/C/D。
9. 教师端扫码并提交识别结果。
10. 大屏实时展示已答人数和选项分布。
11. 教师结束活动。
12. 系统生成课堂报告和 AI 讲评建议。

## 5. 页面需求

### 5.1 教师 Web 端

- 登录页。
- 首页。
- 班级管理页。
- 学生导入页。
- 卡片绑定页。
- 题库管理页。
- 课堂活动页。
- 报告详情页。
- AI 题目生成页。

### 5.2 课堂大屏端

- 活动等待页。
- 题目展示页。
- 答题进度页。
- 匿名统计页。
- 答案解析页。
- AI 讲评建议页。

### 5.3 扫码端 MVP 处理

真实移动端扫码在第二阶段实现。

第一阶段提供“扫码模拟器”，用于提交卡号、选项、识别置信度，先验证课堂业务闭环。

## 6. 验收标准

- 教师能在 2 分钟内创建一次出口检测活动。
- 活动能包含 1 到 3 道单选题。
- 每名学生通过卡号提交一次答案。
- 同一卡号重复提交时，以最新稳定答案为准。
- 大屏不展示学生姓名。
- 教师端能查看学生明细。
- 课堂报告包含正确率、选项分布、未答学生、错因分析和讲评建议。
- AI 生成内容必须标注“AI 生成，教师审核后使用”。
- 系统默认不上传课堂视频和学生图像。
