# 智答课堂 AI 使用说明书

## 一、系统简介

“智答课堂 AI”是一套面向普通教室的课堂即时反馈与智能诊断系统。系统由教师管理端、课堂大屏端、教师手机扫码端和平台后台组成。教师可以在电脑端维护班级、学生和题库，在课堂大屏展示二维码和实时统计；手机端扫码绑定大屏后，可创建课堂、选择题目并通过摄像头采集学生纸质答题卡，实现无学生个人终端条件下的全班即时反馈。

系统访问地址：

- 教师管理端：`https://zhida.foun.fun/teacher/`
- 课堂大屏端：`https://zhida.foun.fun/screen/`
- 教师手机扫码端：`https://zhida.foun.fun/scanner/`
- 平台后台：`https://zhida.foun.fun/admin/`

## 二、适用对象

本系统适用于小学、初中等常规课堂中的随堂检测、出门测、复习检测、课堂投票、学情反馈和课后诊断。学生不需要携带手机、平板或电脑，只需使用统一打印的纸质答题卡。

主要使用角色：

- 任课教师：维护班级和题库，创建课堂，手机扫码采集答案，查看统计与报告。
- 学生：使用纸质答题卡作答。
- 大屏操作人员：在教室电脑或一体机上打开课堂大屏。
- 平台管理员：查看用户、班级、题库、课堂活动等整体数据。

## 三、使用前准备

1. 准备一台可投屏或连接大屏的电脑，打开课堂大屏端。
2. 准备一部教师手机，用微信或浏览器扫码进入教师扫码端。
3. 提前在教师管理端录入班级、学生和题目。
4. 打印并分发学生答题卡，确保每名学生的卡号与系统中学生信息一致。
5. 手机扫码时建议保持光线充足，答题卡尽量平铺，避免严重反光。

## 六、教师手机扫码端使用

### 6.1 扫码绑定大屏

1. 在大屏端显示二维码后，教师用手机微信扫描二维码。
2. 手机打开教师扫码端。
3. 页面顶部显示“已连接大屏，请创建课堂完成绑定”。
4. 如果班级和题目数据已维护，手机端会自动加载班级和题目列表。

> 截图位置：手机端创建课堂页

>
> ![手机端创建课堂页](user-manual/images/mobile-scanner-pairing.png)

### 6.2 创建课堂

1. 在手机端选择班级。
2. 选择学科。
3. 填写教师名称。
4. 在题目列表中选择本次课堂要使用的题目。
5. 点击“生成课堂并开始”。
6. 课堂创建成功后，大屏会自动绑定并显示课堂内容。

注意事项：

- “生成课堂并开始”按钮变亮后才可以点击。
- 如果班级为空，应先到教师端导入班级和学生。
- 如果题目为空，应先到教师端新增或导入题目。

### 6.3 开始扫码采集

1. 课堂创建后，手机端进入扫码采集页面。
2. 点击“开始扫描”。
3. 允许浏览器或微信访问摄像头。
4. 将摄像头对准学生答题卡，可一次扫到多张卡。
5. 系统识别并确认后，会自动上传答案。
6. 大屏同步更新答题进度和统计结果。

扫码建议：

- 手机保持稳定，尽量避免快速晃动。
- 答题卡不要严重遮挡。
- 尽量让答题卡完整进入画面。
- 反光明显时，可调整拍摄角度或距离。
- 如果个别学生未识别，可单独补扫该学生答题卡。

### 6.5 多轮评测

一次课堂可以进行多轮评测。例如第一轮为课前检测，第二轮为讲评后巩固，第三轮为课堂出口测。每轮评测可以选择不同题目，系统会分别记录并生成统计结果。

操作步骤：

1. 完成当前轮评测。
2. 在结果页面选择“新一轮评测”。
3. 输入本轮名称。
4. 选择题目。
5. 开始新一轮扫码。

## 九、答题卡使用说明

每张答题卡包含唯一卡号和方向识别信息。学生通过旋转答题卡表达选项：

- A：将 A 朝上
- B：将 B 朝上
- C：将 C 朝上
- D：将 D 朝上

使用建议：

1. 每名学生固定使用自己的答题卡。
2. 答题卡编号应与系统中学生卡号一致。
3. 答题卡建议打印清晰，避免过度缩放。
4. 可塑封重复使用，但扫码时应避免强反光。
5. 若课堂光线不足，应靠近拍摄或补光。

## 十、常见问题

### 10.1 手机端看不到班级

可能原因：

- 教师端尚未导入班级。
- 手机扫码进入的不是当前大屏二维码。
- 页面缓存了旧状态。
- 当前账号或大屏配对码对应的工作空间不一致。

处理方法：

1. 刷新手机页面。
2. 关闭微信网页后重新扫码。
3. 确认大屏二维码地址为 `https://zhida.foun.fun/scanner/?displayPairCode=...`。
4. 在教师端确认班级和学生已存在。
5. 如仍异常，请重新登录大屏并生成新的配对码。

### 10.2 手机无法打开摄像头

处理方法：

1. 确认浏览器或微信已允许摄像头权限。
2. 退出页面后重新进入。
3. 检查是否有其他应用占用摄像头。
4. 尽量使用系统浏览器或微信内置浏览器的最新版。

### 10.3 扫码识别不稳定

处理方法：

1. 保持手机稳定，避免快速移动。
2. 让答题卡完整进入画面。
3. 调整角度，减少塑封反光。
4. 提高环境亮度。
5. 对未识别学生进行单独补扫。

### 10.4 大屏没有自动进入课堂

处理方法：

1. 确认手机端是从当前大屏二维码进入。
2. 确认手机端已点击“生成课堂并开始”。
3. 刷新大屏页面。
4. 点击大屏“退出课堂”后重新扫码绑定。

### 10.5 题目列表为空

处理方法：

1. 到教师端题库管理中新增或导入题目。
2. 检查题目的学科是否与手机端选择的学科一致。
3. 刷新手机端页面。

## 十一、推荐课堂流程

建议按以下流程使用：

1. 课前在教师端维护班级、学生和题目。
2. 课前打印并分发答题卡。
3. 上课时打开课堂大屏。
4. 教师手机扫描大屏二维码。
5. 手机端选择班级、学科和题目，创建课堂。
6. 学生根据题目旋转答题卡作答。
7. 教师手机扫码采集全班答案。
8. 大屏实时展示统计结果。
9. 教师根据结果即时讲评。
10. 课堂结束后查看报告和 AI 诊断。
