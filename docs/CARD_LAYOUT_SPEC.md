# 智答课堂纸质答题卡版式规范

## 1. 目标

本规范定义 MVP 阶段真实扫码答题卡的纸面版式和码制协议。目标是让教师手机摄像头可以在普通教室环境中稳定识别：

- 唯一答题卡编号，例如 `C001`。
- 学生通过旋转纸卡表达的答案 `A/B/C/D`。
- 识别置信度和多帧确认所需的基础信息。

## 2. 版式

MVP 使用 A6 尺寸单卡，A4 纸每页排 4 张。

| 项目 | 规格 |
| --- | --- |
| 单卡尺寸 | 90mm × 130mm |
| 定位码尺寸 | 54mm × 54mm |
| 定位码位置 | 卡片上半区居中 |
| 安全留白 | 卡片边缘至少 6mm |
| 印刷颜色 | 黑白高对比 |
| 答案标识 | A 在上边，B 在右边，C 在下边，D 在左边 |
| 学生隐私 | 卡面只显示卡号，不显示姓名 |

学生作答时，将所选答案字母转到纸卡顶部朝向教师手机。

## 3. 码制

MVP 使用 `ZhiDaCard v1` 自定义黑白方形码。它不是二维码，设计目标是方便后续 OpenCV 通过方形轮廓和网格解码实现。

### 3.1 网格

- 总网格：7 × 7。
- 外圈 1 格：全黑定位边框。
- 内部 5 × 5：方向锚点和数据位。

内部 5 × 5 的四个角用于方向锚点：

| 位置 | 值 |
| --- | --- |
| 左上 | 黑 |
| 右上 | 白 |
| 右下 | 白 |
| 左下 | 白 |

识别时先找到唯一黑色内角，由此恢复卡片的标准方向；再读取数据位。

### 3.2 数据位

内部 5 × 5 去掉四个角后，剩余 21 个单元按从上到下、从左到右读取：

- 前 10 位：卡号数字部分，支持 `1-1023`。
- 接下来 4 位：校验位，算法为 `(id ^ (id >> 4) ^ (id >> 8)) & 0x0f`。
- 最后 7 位：固定版本位 `1010110`。

例如 `C001` 的数字部分为 `1`。

## 4. 答案方向映射

摄像头识别到码以后，应得到标准方向相对画面顶部的旋转角。

| 学生选择 | 卡片顶部朝向 | 旋转含义 |
| --- | --- | --- |
| A | 标准顶部朝上 | 0° |
| B | 标准右侧朝上 | 90° |
| C | 标准底部朝上 | 180° |
| D | 标准左侧朝上 | 270° |

MVP 可以把角度归一到最近的 90°，再映射为选项。

## 5. 识别流程

1. 摄像头取帧。
2. 灰度化、二值化。
3. 查找近似正方形黑色外框轮廓。
4. 透视矫正为正方形码图。
5. 采样 7 × 7 网格。
6. 校验外圈黑边。
7. 根据内部黑色方向锚点确定标准方向。
8. 解码 10 位卡号和 4 位校验。
9. 根据旋转角转换 `A/B/C/D`。
10. 多帧确认后提交 `{ cardCode, selectedOption, recognitionScore, recognizedAt }`。

## 6. 多帧确认建议

- 同一 `cardCode` 连续 3 帧识别结果一致后确认。
- 单帧置信度低于 `0.75` 时不确认。
- 2 秒内同一卡号多次确认时，以最新方向为准。
- 上传前按现有批量接口聚合，减少网络抖动。

`services/cv-service` 已提供 `FrameConfirmor` 实现上述确认逻辑。确认后输出结构可直接转换为后端批量答题接口中的单条答案。

## 7. 图像采样

`services/cv-service` 已提供 `sampleMarkerCells`。当摄像头或后续 OpenCV 流程定位到码的四个角以后，它会从灰度图中采样 7×7 网格，再交给 `decodeObservedCells` 完成卡号和答案方向解码。

`locateMarkerCorners` 提供 MVP 级定位能力：在白底画面中查找最大黑色方形区域并返回四角。它适合当前打印测试卡和开发验证；真实课堂复杂背景下，后续应替换为 OpenCV 轮廓检测和透视矫正。

`ScanFramePipeline` 已将定位、采样、解码和多帧确认串联起来。输入单帧灰度图后，它会返回单帧解码结果；连续 3 帧一致后，返回可上传的确认答案。

`AnswerBatchCollector` 用于收集确认答案，并生成后端 `/api/sessions/{sessionId}/answers/batch` 所需的请求体。同一卡号重复确认时，只保留最新答案。

`ScanSession` 将单帧扫码管线和批量上传聚合器组合起来。移动端接入摄像头后，可以为每个活动题目创建一个 `ScanSession`，持续传入视频帧，并周期性调用 `drainUploadPayload()` 上传。

`apps/scan-debug` 提供开发期扫码调试台。它可以用合成帧模拟纸卡识别，展示单帧解码、多帧确认、待上传 payload，并在粘贴真实 `sessionId` 和 `questionId` 后调用后端批量答题接口。

